Оффлайновая система проверки задач.

Структура директорий.
checkit.bat - программа для проверки под Windows.
checkit.sh - программа для проверки под Unix/Linux.
task - директория с задачами.
task/if/if_1 - файлы задачи if_1 из раздела if. Файлы с расширением dat - входные данные, с расширением ans - ожидаемый вывод задачи.
task/if/list.txt - краткий список задач раздела с рекомендуемым порядком решения задач.
solutions - директория с исполняемыми файлами.
solutions/if/if_1.c - этот файл будет взят под Unix/Linux и откомпилирован перед запуском программы. Предполагается, что тут вы решаете задачи.
solutions/if_1.exe - исполняемый файл. Создается автоматически под Unix/Linux. Под Windows система проверки ожидает файл с этим именем в данной директории для проверки тестов задачи if_1.
ВНИМАНИЕ, ПОД WINDOWS ИСПОЛНЯЕМЫЕ ФАЙЛЫ ПРИДЕТСЯ КОМПИЛИРОВАТЬ И СОБИРАТЬ САМИМ, И САМИМ ЖЕ КОПИРОВАТЬ НУЖНЫЙ ФАЙЛ В ЭТУ ДИРЕКТОРИЮ.

Запуск под Unix/Linux.
1. Перейдите в директорию с файлом checkit.sh
2. ./checkit.sh if 1
   Для запуска на компиляцию и проверку задачи if_1 из раздела if.
Возможные проблемы:
1. Проблема: Файл checkit.sh не запускается.
1. Решение: Возможно, у файла нет прав на исполнение. Выполните chmod +x checkit.sh
2. Проблема: вы компилируете не gcc и вообще в оболочке, система проверки задач не может запустить файл на компиляцию.
2. Решение: Придется компилировать в нужный exe файл самим и класть его в нужную директорию solutions.

Запуск под Windows.
1. Проблема: Система проверки задач не может откомпилировать ваши файлы.
1. Решение: Это известная проблема. Не может. Придется компилировать самим и копировать нужный exe файл с тем именем, которое ожидает система, в директорию solutions.
